# 구현 명세

## 목적

Jira에 익숙하지 않은 팀원이 자연어로 할 일만 적으면 서버 agent가 Jira MCP로 티켓을 생성한다. 이 서비스는 Jira를 대체하지 않고 티켓 생성만 전담한다.

## 확정 결정

- 서비스 이름: `할 일 올리기`
- 로그인: 사전 등록 이메일/비밀번호
- 세션: 30일
- Jira 프로젝트: `S15P11A705`
- 타입: Task
- 담당자: 로그인 사용자의 필수 `jira_account_id`
- 생성 주체: 서버의 현재 Atlassian MCP 인증
- 스프린트·parent·story point: 설정하지 않음

## 사용자 흐름

로그인 → 자연어 입력 → 티켓 만들기 → 생성된 티켓 링크 확인.

화면은 로그인과 메인 두 개뿐이다. 메인 안에서는 페이지를 이동하지 않고 상태만 바뀐다.

## 생성 규칙

- 한 줄은 기본적으로 한 Task다.
- 한 문장에 명확한 별도 작업이 여러 개일 때만 분리한다.
- 완료 항목, 일정·회의, 메모·생각은 제외하고 내부 로그에 남긴다.
- summary는 원문을 한 줄 작업 형태로만 다듬는다.
- description은 명시된 맥락만 사용하며 없으면 비운다.
- 인수조건과 사실을 지어내지 않는다.
- 자동 재시도하지 않는다.

## 만들지 않는 것

회원가입, 비밀번호 찾기, 프로젝트/타입/담당자 선택, 에픽/하위 작업, story point, 스프린트 배정, 생성 전 편집, 이월 감지, Markdown 구조 파싱, 날짜별 history, Jira 상태 조회, 티켓 검색·수정·삭제, 관리 화면.

## 오류와 보존

- 원문은 agent 실행 전에 private SQLite에 저장한다.
- idempotency key로 중복 요청을 막는다.
- Jira 내부 오류를 화면에 노출하지 않는다.
- 오류 원문·입력 원문·사용자 이름·시각은 Mattermost Incoming Webhook으로 리드에게 보낸다.
- 부분 성공은 성공 티켓을 반환하고 실패 수를 알린다.
- 생성 결과 목록은 브라우저 메모리에만 유지한다.
